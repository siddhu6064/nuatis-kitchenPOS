6. apps/pos-admin/src/app/cash/page.tsx — replace placeholder; server component:
   - Fetches GET /v1/cash/sessions/current?location_id={selected}
   - If 404: render <NoCashSession /> empty state with "Start a shift via the cashier terminal" message
   - If session exists: render <CashSessionView session={session} />

7. apps/pos-admin/src/components/cash/CashSessionView.tsx (client):
   - Header: shift opened by {staff name}, since {opened_at relative}, opening float ${X.XX}
   - Live running total (recomputes from events + opening float every 10s):
     * Cash sales: +$X.XX
     * Cash refunds: −$X.XX
     * Pay-ins: +$X.XX
     * Pay-outs: −$X.XX
     * Expected total: $Y.YY
   - Event timeline: list of cash_events for this session, newest first, each row: time + type badge + amount + reason + staff name
   - Actions:
     * "Add cash event" button → opens dialog with type dropdown (pay_in, pay_out, no_sale), amount, reason
       - Submitting pay_out or no_sale triggers manager PIN modal first
       - On success: refetch session
     * "Close shift" button → opens confirmation modal asking for closing actual amount (cents → dollars input)
       - Compares to expected, shows variance
       - Confirms → calls POST /v1/cash/sessions/:id/close

8. apps/pos-admin/src/components/cash/CashEventDialog.tsx (client):
   - Form: type select, amount in dollars, reason text
   - On submit: POST /v1/cash/sessions/:id/events
   - If 403 manager_pin_required: show ManagerPinModal, retry with manager_pin in body

9. apps/pos-admin/src/components/cash/ManagerPinModal.tsx (reusable):
   - 4-digit PIN input with auto-focus and large tap targets
   - On submit: returns the entered PIN to caller (caller adds to API request body)
   - Triggered by 403 manager_pin_required from any API call
   - Note: this component is reusable for void/refund flows too

10. apps/pos-admin/src/components/cash/CloseShiftDialog.tsx (client):
    - Form: closing actual ($)
    - Computes variance = closing_actual_cents - expected_cents inline
    - Color-coded: green (variance == 0), yellow (|variance| ≤ $5), red (|variance| > $5)
    - Confirm button only enabled when amount entered
    - On submit: POST /v1/cash/sessions/:id/close → toast success, navigate to /cash showing closed-state empty page

11. Add api-client wrappers in apps/pos-admin/src/lib/api/cash.ts:
    - getCurrentSession(location_id)
    - logCashEvent(sessionId, params, managerPin?)
    - closeSession(sessionId, closingActualCents)
