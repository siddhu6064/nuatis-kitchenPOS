SECTION 2 — Tablet prototype API client
═══════════════════════════════════════

6. artifacts/nuatis-pos/src/lib/api/ — new folder with these files:

7. artifacts/nuatis-pos/src/lib/api/client.ts:
   - Reads VITE_API_BASE_URL from import.meta.env (default http://localhost:3002)
   - Reads JWT from sessionStorage key 'pos.jwt' (NOT localStorage — sessionStorage clears on tab close, more appropriate for cashier session)
   - Generic fetcher: request<T>(path, options) → Promise<T>
     - Sets Authorization: Bearer <token> if JWT exists
     - Content-Type: application/json
     - Throws ApiError on non-2xx (with parsed { error: { code, message } } body)
     - Returns parsed JSON typed as T
   - Helpers: get<T>, post<T,Body>, patch<T,Body>, del(path)
   - Export setJwt(token) and clearJwt() helpers

8. artifacts/nuatis-pos/src/lib/api/auth.ts:
   - signInWithPin(tenant_id, location_id, pin) → calls POST /v1/auth/pin → stores token via setJwt → returns parsed response
   - signOut() → clears JWT from sessionStorage
   - Uses types imported from a local copy of pos-shared schemas (since the prototype is in a different workspace — see section 4)

9. artifacts/nuatis-pos/src/lib/api/menu.ts:
   - getMenuTree() → GET /v1/menu/tree → returns nested categories+items+modifiers
   - Parses response with the local Zod schemas; throws on shape mismatch

10. artifacts/nuatis-pos/src/lib/api/orders.ts:
    - createOrder(location_id, staff_id) → POST /v1/orders
    - addOrderItem(order_id, menu_item_id, quantity, modifiers) → POST /v1/orders/:id/items
    - voidOrderItem(order_id, item_id) → DELETE /v1/orders/:id/items/:item_id
    - sendToKitchen(order_id) → POST /v1/orders/:id/send-to-kitchen
    - checkout(order_id) → POST /v1/orders/:id/checkout
    - createPayment(order_id, method, tip_cents) → POST /v1/orders/:id/payments
