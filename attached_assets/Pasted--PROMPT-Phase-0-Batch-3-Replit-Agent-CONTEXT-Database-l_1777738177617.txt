9. apps/pos-api/src/middleware/error-handler.ts:

   - Logs error with logger
   - Returns { error: { code: string, message: string } } JSON, never leaks stack traces in production
   - 500 default, but reads err.statusCode if set

10. apps/pos-api/src/middleware/request-id.ts:
    - Generates request ID (uuid v4) per request, attaches to req.id, sets X-Request-Id response header
    - Used by pino-http for structured logging correlation

11. apps/pos-api/src/routes/health.ts:
    - GET /v1/health
    - Returns { status: 'ok', supabase: 'connected' | 'disconnected' | 'not_configured', uptime_ms: number, version: string (read from package.json), timestamp: ISO8601 }

    - Never returns 500 — health endpoint should always respond 200, status field carries the truth
