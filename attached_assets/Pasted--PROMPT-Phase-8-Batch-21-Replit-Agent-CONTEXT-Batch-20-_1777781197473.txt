SCOPE — BUILD THIS
1. Wire redactSensitive into every pino instance — choose either (a) wrap pino log methods
   in a thin module that calls redactSensitive on the merging object before delegating to
   pino, or (b) configure pino's built-in `redact` option against a path list derived from
   SENSITIVE_KEYS exported from the existing redact.ts module. Either approach is acceptable
   so long as the acceptance test below passes. Do NOT duplicate the SENSITIVE_KEYS list.

2. GET /v1/audit-log endpoint in pos-api — paginated (limit default 50, max 200). Query
   params: action_type (string), staff_id (uuid), from (ISO date), to (ISO date), cursor
   (opaque string). Owner+manager only; tenant_id from req.auth. Returns
   { entries: [...], next_cursor: string|null }. Each entry: id, occurred_at, action_type,
   staff_id, staff_name (joined), target_type, target_id, payload, ip_address.

3. GET /v1/audit-log.csv — same query params, CSV response. Owner-only (return 403 for
   manager). Use the existing CSV pattern from /v1/reports/end-of-day.csv.

4. Admin /audit page — server component with filters driven by URLSearchParams: action_type
   dropdown (populate from distinct values returned in the API), staff dropdown (from
   /v1/staff), from + to date pickers. Table with the entry columns above. Click a row to
   expand a payload view showing the JSON in a monospace block.

5. Owner-only "Export CSV" button — visible to owner role, hidden for manager. Posts to
   the .csv endpoint with current filter state.

6. Admin nav — add "Audit" link between Devices and Settings using a lucide-react icon
   (ScrollText or History).
