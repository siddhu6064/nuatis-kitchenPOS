SECTION 6 — Webhook handler
═══════════════════════════════════════

9. apps/pos-api/src/routes/stripe/webhook.ts:
   - POST /v1/webhooks/stripe — NO auth (Stripe-signed, public)
   - CRITICAL: must use raw body for signature verification — register before express.json() middleware OR use express.raw({ type: 'application/json' }) on this route only
   - Use stripe.webhooks.constructEvent(body, sigHeader, STRIPE_WEBHOOK_SECRET) to verify
   - On signature failure: 400 with no body (do NOT echo error details)
   - Idempotency: use Stripe event.id; check audit_log for prior 'stripe_webhook' entry with same event.id; if exists, return 200 OK (already processed)
   - Handle event types:
     * account.updated → look up tenant by stripe_account_id; update charges_enabled, payouts_enabled, requirements_currently_due; audit log
     * payment_intent.succeeded → look up payment by stripe_payment_intent_id; update payment status='succeeded'; update order status='paid'; record card_brand + card_last4 + stripe_charge_id from charge object; audit log; broadcast to terminal Realtime channel `payment:{order_id}` so the prototype's Tap-to-Pay screen advances
     * payment_intent.payment_failed → update payment status='failed', error message; broadcast failure to terminal channel
     * charge.refunded → look up payment by stripe_charge_id; insert/update refunds row; audit log
   - Always returns 200 with empty body (Stripe expects 2xx)
   - Mount in apps/pos-api/src/index.ts BEFORE express.json() — important ordering

10. apps/pos-api/src/lib/idempotency.ts (small helper):
    - hasProcessedEvent(event_id) → checks audit_log for { action: 'stripe_webhook', payload->>event_id: X }
    - markEventProcessed(event_id, type, payload) → writes audit_log row
    - Used by webhook to dedupe
