SECTION 5 — pos-api routes

11. apps/pos-api/src/routes/receipts/send.ts:
    - POST /v1/orders/:id/receipts — any auth (session or terminal); body = SendReceiptRequestSchema
    - Validates order is paid (cannot send receipt for unpaid order)
    - Upserts contact: lookup by phone (if provided) or email (if provided) within tenant; create if not found
    - If sms_opt_in true: update contact.sms_opt_in=true, sms_opt_in_at=now(), sms_opt_in_text=body.sms_opt_in_text || 'Yes, send me text receipts', sms_opt_in_ip=req.ip
    - Sign receipt token; build receipt_url = `${env.RECEIPT_BASE_URL}/r/${token}`
    - If email present: enqueue receipt-email job with { tenant_id, order_id, to: email, receipt_url }
    - If phone present AND contact.sms_opt_in true: enqueue receipt-sms job with { tenant_id, order_id, to: phone, receipt_url }
    - Returns { jobs_enqueued: [...], receipt_token }
    - Audit log action='receipt_send' with channels in payload

12. apps/pos-api/src/routes/receipts/view.ts:
    - GET /r/:token — public route (NO auth required) — but verify token signature
    - Decodes token → fetches order + items + payment + tenant + location
    - Returns minimal HTML page (not JSON) with the receipt — owner brand visible, order details, totals, tip, last4 if card payment
    - Use a simple template literal; no React SSR for this batch
    - Cache headers: private, max-age=86400 (receipts are immutable)
    - 404 if token invalid or order not found
