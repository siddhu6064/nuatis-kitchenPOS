SECTION 2 — Auth.js v5 setup
═══════════════════════════════════════

8. apps/pos-admin/src/auth.ts (or src/lib/auth.ts depending on convention):
   - Auth.js v5 config with Credentials provider
   - authorize() function:
     * Calls POST {POS_API_URL}/v1/auth/sign-in with { email, password }
     * If 200 with token: returns user object { id: response.user.id, email, name: response.user.email, tenant_id: response.user.tenant_id, role: response.user.role, posJwt: response.token }
     * If 401: returns null (Auth.js shows error)
   - JWT callback: copies tenant_id, role, posJwt onto the JWT
   - Session callback: copies them onto session.user
   - secret: NEXTAUTH_SECRET
   - pages: { signIn: '/sign-in' }
   - Export NextAuth handlers, signIn, signOut

SECTION 4 — pos-api: sign-up endpoint
═══════════════════════════════════════

14. apps/pos-api/src/routes/onboarding/sign-up.ts:
    - POST /v1/onboarding/sign-up — NO auth required (public)
    - Body schema (add to packages/pos-shared/src/auth.ts as SignUpRequestSchema):
      * business_name: string min 1 max 100
      * vertical: enum ['cafe', 'restaurant'] (MVP launch verticals only — locked decision)
      * full_name: string min 1 max 100
      * email: lowercase email
      * password: string min 8
      * timezone: string optional (default 'America/Chicago')
      * terms_accepted: literal true
    - Server-side flow (single transaction where possible):
      1. Check no existing tenant has owner staff_member with this email — if conflict, 409 with code 'email_taken'
      2. INSERT tenants row with (id, name=business_name, vertical, timezone)
      3. INSERT locations row (auto-default location, name=business_name, sales_tax_bps=825, business_hours=null)
      4. bcrypt-hash password (cost 12)
      5. INSERT staff_members row with (tenant_id, location_ids=[location.id], full_name, email, role='owner', password_hash, pin_hash=null)
      6. Audit log entry: action='tenant_created', target_type='tenant', target_id=tenant.id, payload={vertical, business_name}
      7. Return 201 with { tenant_id, location_id, owner_staff_id, message: "Account created. Check your email to confirm." } — but DON'T actually send email this batch (deferred)
    - Validate via SignUpRequestSchema
    - Mount router at /v1/onboarding/sign-up in pos-api/src/index.ts
    - Add tests: valid sign-up creates all 3 rows, duplicate email returns 409, invalid vertical returns 400

15. Add SignUpRequestSchema and SignUpResponseSchema to packages/pos-shared/src/auth.ts; rebuild pos-shared.
